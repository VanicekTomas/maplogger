CartoLogger Extension

Case-study note: This CartoLogger tab is currently tailored to one case study (IPAtlas). If you want to use it for another study or a different workflow, custom adaptation is possible; please contact the author via the project repository.

This tab extends the original MapLogger idea with support for CartoLogger CSV exports. CartoLogger is a Chrome extension that records map interactions (clicks, pans, zooms, and URL modifications) and exports logs for behavioural analysis.

The analysis here focuses on URL-based interaction traces, especially url_change events, and explains what changed between consecutive URLs (zoom, map layer, coordinates, and related parameters).

Tool source: CartoLogger GitHub repository. Example study use: ICCGIS 2024 paper (PDF).

2) Converted interaction model (task_ID-based)

Task_ID filter is inclusive. Leave both fields empty to include all task_ID values.

The converter renames session to participant_ID, computes task_ID from the last url_change in each original task, removes rows with pointerup and mouse_wheel_end, renames mouse_wheel_start to zoom, derives columns parameter, zoom, map, click, projection, and removes val from the exported CSV.
